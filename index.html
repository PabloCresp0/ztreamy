Ztreamy 0.2.1 has been released on 2014-12-01 (download). This version depends on a specific version of the tweepy library, because their latest version seems to be unistalable. There aren't any other changes with respect to version 0.2.

Ztreamy 0.2 has been released on 2014-09-18 (download). This version includes some fixes and facilitates the development of JavaScript clients by streaming events with JSON-LD syntax when the client requests that media type in the Accept HTTP header and by using CORS for facilitating cross-site JavaScript requests.

Some fixes to the protocol (nicer media types for Ztreamy events and streams, mandatory CRLF line delimiters, compression signaled with the standard Content-Encoding HTTP header) make it incompatible with the previous versions of Ztreamy. Once these changes have been introduced, the protocol is planned to be stable for the next versions.

Introduction

Ztreamy is a prototype implementation of a framework for publishing streams of semantically-annotated data (e.g. readings from sensores) through on the Web, built on top of the Tornado Web server. The framework provides services that are specific to events described with RDF, although any other kinds of events can be transported as well. The platform provides scalability through replication: streams can be repeated from other servers. Streams can also be aggregated and filtered.

A thorough description and evaluation of Ztreamy has been published at the Journal of Web Semantics. Its preprint is publicly available for download. If you want to cite Ztreamy from a research publication or report, please use the following bibliographic reference:

  • Jesus Arias Fisteus, Norberto Fernandez Garcia, Luis Sanchez Fernandez, Damaris Fuentes-Lorenzo, Ztreamy: a middleware for publishing semantic streams on the Web. Journal of Web Semantics, volume 25, pp. 16-23 (2014). doi:10.1016/j.websem.2013.11.002. Access the preprint version.
